What is this?

The Expert Network Search Copilot is a backend API for searching professional candidate profiles using natural language. No SQL, no manual filters — describe who you need in plain English.

ModeBest forEndpointSpeed
Expert SearchFinding specific expert profilesPOST/chat0.5–1.5s
SQL AnalyticsCounts, distributions, aggregationsPOST/query3–8s
Expert ProfileFull CV for a specific candidateGET/expert/{id}<0.3s

How it works

1

Query understanding

Detects signals: geography, seniority, industry, experience threshold, languages.

2

Semantic vector search

Converts your query to an embedding and matches against 10K+ profiles via cosine similarity in Pinecone.

3

Structured re-ranking

Adjusts scores using deliberate signal weights. Geography match +0.20; below experience threshold −0.15.

4

Explainable results

Every result includes a match_explanation with exact signals. Transparent enough to show a client.

Quick Start

Health check

curl http://localhost:8001/health
# → {"status": "ok"}

First search

curl -X POST http://localhost:8001/chat \
  -H "Content-Type: application/json" \
  -d '{"query":"Find pharma regulatory experts in Saudi Arabia","top_k":5}'
💡
Open http://localhost:8001/docs for the interactive Swagger UI — try every endpoint without curl.

Multi-turn Search

Every response includes a conversation_id. Pass it back to make follow-up queries that reference prior results.

# Turn 1
curl -X POST http://localhost:8001/chat \
  -d '{"query":"Find pharma regulatory experts in the Middle East"}'
# → "conversation_id": "abc-123-..."

# Turn 2 — narrow
curl -X POST http://localhost:8001/chat \
  -d '{"query":"Filter to only Saudi Arabia","conversation_id":"abc-123-..."}'

# Turn 3 — narrow further
curl -X POST http://localhost:8001/chat \
  -d '{"query":"Which have 10+ years?","conversation_id":"abc-123-..."}'
💡
Follow-up triggers detected automatically: those, them, filter, narrow, only, same, above. Start broad, then narrow progressively.

Hard Filters

Filters apply before vector search. Only matching candidates are considered at all.

By country

{"query":"Data scientists","filters":{"country_name":{"$eq":"Saudi Arabia"}}}

By minimum experience

{"query":"Pharma consultants","filters":{"years_of_experience":{"$gte":10}}}

Multiple constraints (AND)

{"query":"Regulatory experts","filters":{"$and":[{"country_name":{"$eq":"Saudi Arabia"}},{"years_of_experience":{"$gte":8}}]}}
⚠️
Very restrictive filters may return fewer results than top_k. If that happens, relax the filter and encode the preference in the query text instead.

Available filter fields

country_name · city_name · years_of_experience · current_title · gender · country_code

SQL Analytics — /query

For counting, aggregating, and distributing data. Internally converts your question to SQL using a LangGraph agent.

"How many candidates speak Arabic at a Native level?"
"Top 10 most common skills across all candidates"
"Distribution of candidates by years of experience in 5-year buckets"
"Which countries have the most candidates with a PhD?"
curl -X POST http://localhost:8001/query \
  -H "Content-Type: application/json" \
  -d '{"question":"How many candidates hold an MBA?","include_sql":true}'

Hybrid Mode

The /chat endpoint automatically routes analytical sub-queries (containing "how many", "count", "top") to the SQL agent in parallel with vector search — so you get both a ranked shortlist and analytical context in one call.

curl -X POST http://localhost:8001/chat \
  -d '{"query":"Find top pharma experts in Saudi Arabia -- how many exist in total?","top_k":5}'
# Response includes both "results" (ranked shortlist) and "sql_answer" (total count)

Troubleshooting

Fewer results than top_k

Filters too restrictive. Relax the filter and encode the preference in the query text.

Top result doesn't seem relevant

Check match_explanation.signals. Add a country name or seniority level to the query to activate the corresponding signal.

/query returns "No results found"

Check spelling of proper nouns. Database uses: "Saudi Arabia" not "KSA"; "Pharmaceutical" not "pharma".

422 Unprocessable Entity

Validation error — query shorter than 3 chars, top_k outside 1–50, or malformed JSON.

NotFoundError: anthropic/claude-3.5-sonnet

Deprecated model ID. Set LLM_MODEL=anthropic/claude-sonnet-4.6 in your .env.

Glossary

TermDefinition
EmbeddingA numerical vector capturing semantic meaning. Similar texts have similar embeddings.
Vector searchFinding the most similar candidates to a query by comparing embedding vectors.
Re-rankingAdjusting raw similarity scores using structured signals (geography, experience, seniority, industry, language).
PineconeThe managed vector database used to store and search candidate embeddings.
Text-to-SQLConverting a natural language question into SQL executed against PostgreSQL.
Precision@5Fraction of top-5 results that are actually relevant. The primary quality metric.
Conversation IDSession token linking follow-up queries to prior search results.
